# M0 — Scaffolding & setup (in simple words)

*(Written after the fact — this stage was done before the learning folder existed.)*

## What this stage is

Before writing real code, we made a clean "home" for the project.

## What we did

1. **Folders.** We made empty folders: `app/` (the bot's brain), `host/` (the
   dashboard app), `data/` (the database file lives here), `tests/` (the tests).

2. **A list of tools.** `requirements.txt` is a shopping list of the Python
   packages we need. We wrote each one with its exact version number, so the
   project installs the same way every time, on any computer.

3. **A private sandbox.** A "virtualenv" (`.venv`) is a private copy of Python
   just for this project. The packages we install go there, not into the whole
   computer. This stops projects from stepping on each other.

4. **Secrets stay out of git.** Passwords and API keys go in a file called
   `.env`. We told git to **ignore** that file, so secrets are never uploaded.
   We did commit `.env.example` — the same file but with the values blanked
   out — so anyone can see *which* secrets are needed without seeing them.

5. **Accounts.** On Meta's website we created an app, turned on WhatsApp, and
   copied four values: the phone number id, an access token, the app secret,
   and a verify token we invented. We also got an OpenAI key.

6. **A tunnel.** Our computer is hidden behind a home router, so Meta cannot
   reach it. A tunnel (ngrok) gives us a public web address that forwards to
   our computer. We reserved a fixed address so it doesn't change every restart.

7. **A settings loader.** `app/config.py` reads all those secret values from
   `.env` into one tidy `Settings` object. If a secret is missing, the app
   refuses to start right away — better to fail loudly at the start than
   mysteriously later.
