# M2 — Phone handling (in simple words)

*(Written after the fact — this stage was done before the learning folder existed.)*

## What this stage is

One phone number can be written many ways: `054-123-4567`, `0541234567`,
`+972541234567`, `972541234567`. The computer sees four different strings. We
need them all to become **one** exact form, or the bot will never match a
guest's reply to their invitation.

## What we did

1. **Picked one canonical form: E.164.** That is the international format —
   `+`, country code, number, no spaces or dashes. Example: `+972502345678`.
   Every number is converted to this form **before** it is saved or compared.

2. **`app/phone.py` — one function, `normalize_phone`.** It uses Google's
   `phonenumbers` library (the same brain behind Android's dialer). The logic:
   - Starts with `+`? Trust it — the country code is right there.
   - No `+`? First try reading it as a **local Israeli** number (`054…`).
   - That failed? Try adding `+` in front — this handles WhatsApp's `wa_id`,
     which is the full international number with the `+` stripped
     (`972502345678`).
   - Nothing worked, or the number isn't real? **Reject it.** We never store a
     junk number.

3. **Tests.** We check that all the different spellings of the same number come
   out identical, and that garbage like `"hello"` is refused. One gotcha we
   learned: the library knows which number ranges *really exist*, so a made-up
   number can fail validation even if it "looks" fine.
